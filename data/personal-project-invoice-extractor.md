## 3. Intelligent Expense Verification & Bill-Tracking Engine

*   **System Overview & Pipeline Architecture:** Developed an automated personal accounting and expense-tracking dashboard that interfaces directly with communication layers via the Gmail API to aggregate financial overhead. Engineered a dual-engine parser that extracts transaction metadata, due dates, billing periods, and values from raw email bodies (plain text and HTML processed via BeautifulSoup) and PDF statements (compiled using `pypdf`). Serves data to a single-page HTML5 analytics dashboard via a FastAPI and Uvicorn backend.
*   **AI Refinement & Categorization Layer:** Integrated a Large Language Model (LLM) classification gate using `gemini-2.5-flash`. The model batch-inspects email subjects, sender headers, and content payloads to distinguish actionable utility invoices, recurring subscriptions, and receipts from administrative alerts, terms updates, and support logs with high precision.
*   **Dynamic Feedback & AI Rule Generation:** Engineered an active-learning user feedback loop. When a user marks an item as declined or miscategorized on the dashboard, the engine queries Gemini to analyze the failure context (subject, sender, and user-provided rejection reason). Gemini synthesizes a generalized, date/number-excluded keyphrase exclusion rule which is cached locally in JSON. Future incoming messages are filtered instantly using these localized rules, avoiding LLM token overhead and reducing latency.
*   **Temporal Consolidation & Reminder Matching:** Implemented a matching heuristic that detects payment reminders or late notices and consolidates them with their parent invoice record. The algorithm matches items by sender grouping (normalizing domains like Cloudflare, AWS, or Google Cloud), exact original currency/amount, and a chronological window of 25 days. It displays unified logs (tracking reminder counts and last-received dates) to prevent ledger duplication.
*   **Multi-Currency Normalization Engine:** Integrated a historical currency conversion layer that calls the Frankfurter API to normalize amounts from various currencies (USD, CAD, EUR, INR) to a user's default reporting currency (e.g., INR) based on the exact transaction/invoice date. Implements persistent JSON caching for exchange rates with structured fallbacks for offline resilience.
*   **Key Learnings & Impact:** 
    *   *AI API Cost & Latency Optimization:* Learned to balance LLM workloads by utilizing local, cached rule heuristics compiled by AI, resulting in zero-token local execution for subsequent matching emails.
    *   *Robust Data Normalization:* Addressed formatting variances in invoice numbers, dates, and amounts using pre-compiled regex arrays coupled with context-window heuristics to ensure high extraction reliability without relying purely on LLMs.
    *   *User-in-the-Loop AI:* Demonstrated how low-friction user actions (accept/decline) can drive background AI rule adaptation, creating a highly customized categorization system.
